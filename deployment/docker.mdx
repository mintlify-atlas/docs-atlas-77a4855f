The Stremio Web repository ships a `Dockerfile` that produces a minimal, self-contained image. It uses a multi-stage build to keep the final image small: only the compiled assets and a lightweight Express server end up in the image you run.

## Build stages

The Dockerfile defines four stages:

| Stage | Base | Purpose |
|---|---|---|
| `base` | `node:20-alpine` | Shared foundation — enables pnpm via Corepack, installs git, creates the working directory |
| `app` | `base` | Installs all dependencies and runs `pnpm build` to produce the `build/` directory |
| `server` | `base` | Installs only `express@4` — no dev dependencies |
| _(final)_ | `base` | Copies `http_server.js`, `node_modules/` from `server`, and `build/` from `app` |

The final image contains no source code, no dev dependencies, and no build tooling.

## Build the image

Run the following command from the repository root:

```bash
docker build -t stremio-web .
```

The build process:
1. Installs all pnpm dependencies from the lockfile (`--frozen-lockfile`).
2. Runs `pnpm build` (Webpack production build).
3. Assembles the final image with only the Express server and compiled assets.

<Tip>
The Webpack build embeds the current Git commit hash into asset paths. Make sure you're building inside a Git repository with at least one commit.
</Tip>

## Run the container

```bash
docker run -p 8080:8080 stremio-web
```

The container listens on port **8080**. Once started, the server logs:

```
Server listening on port: 8080
```

Open `http://localhost:8080` in your browser to access the app.

## Docker Compose

```yaml docker-compose.yml
services:
  stremio-web:
    image: stremio-web
    build: .
    ports:
      - "8080:8080"
    restart: unless-stopped
```

Build and start with:

```bash
docker compose up --build
```

## Environment variables

The following variables are baked into the client bundle at build time by Webpack's `EnvironmentPlugin`. Pass them as `--build-arg` values or `ARG`/`ENV` pairs if you extend the Dockerfile.

| Variable | Default | Description |
|---|---|---|
| `SENTRY_DSN` | `null` | Sentry DSN for error reporting. Leave unset to disable Sentry. |
| `SERVICE_WORKER_DISABLED` | `false` | Set to `true` to skip registering the Workbox service worker. |
| `DEBUG` | `false` in production | Enables debug output in the client. Automatically `false` for production builds. |
| `VERSION` | Value from `package.json` | Application version string embedded in the bundle. |

<Note>
These variables are embedded at **build time**, not at runtime. Changing them requires rebuilding the image.
</Note>

## Caching behaviour

The Express server (`http_server.js`) sets HTTP cache headers on all responses:

| Resource | `Cache-Control` max-age |
|---|---|
| `index.html` | 7,200 seconds (2 hours) |
| All other assets | 2,629,744 seconds (~1 month) |

Long-lived asset caching is safe because asset filenames include the Git commit hash, so each new build produces unique URLs.

## Health check

Add a Docker health check to verify the server is responding:

```yaml docker-compose.yml
services:
  stremio-web:
    image: stremio-web
    build: .
    ports:
      - "8080:8080"
    restart: unless-stopped
    healthcheck:
      test: ["CMD", "wget", "-qO-", "http://localhost:8080"]
      interval: 30s
      timeout: 5s
      retries: 3
      start_period: 10s
```

`wget` is available in the `node:20-alpine` base image without any additional packages.

## Production considerations

<Warning>
Do not expose the container directly to the public internet without a reverse proxy. Use nginx, Caddy, or a cloud load balancer to terminate TLS.
</Warning>

**Reverse proxy example (nginx):**

```nginx
server {
    listen 443 ssl;
    server_name your-domain.example.com;

    location / {
        proxy_pass http://127.0.0.1:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**Run as non-root:** The default `node:20-alpine` image runs as root inside the container. For stricter environments, add a `USER node` instruction to your Dockerfile before the `CMD`.

**Resource limits:** The Webpack build is CPU- and memory-intensive. Allocate at least 1 GB of memory to the build container if running in a constrained CI environment.
